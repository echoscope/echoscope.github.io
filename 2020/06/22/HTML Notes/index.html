HTML Introduction

HTML stands for Hypertext Markup Language.

Key points to understand HTML

  1. HTML is not a computer programming language.
  2. As a package of tags, HTML is used to describe text in a structured way.
  3. HTML structured text file usually has a .html filename suffix.
  4. What web browser, viewer, parser … do is to read HTML file and ouput the structured page so user can read joyfully.

Key points to understand HTML tag

  1. HTML is a pacakge of markup tags that can be used in plain text to construct HTML page.
  2. HTML tag usually appears in paris, first one is called opening tag and the second closing tag.
  3. HTML tag with their inner content is called a HTML Element. Example: <p> Content </p>
  4. Single HTML tag is called empty tag or empty element. Example: <br/>

DOCTYPE

As other software standards, HTML standards has its own development progress. The latest standard is HTML 5.
In every HTML Document, you would see a <!DOCTYPE> tag at the very beginning of the Document. The tag indicates the HTML standard the document is using.
The HTML 5, will have a <!DOCTYPE html> as DOCTYPE.
For now there are still many web pages using HTML 4.0.1 or XHTML 1.0. Their Doctype are as follows:
HTML 4.0.1:
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
XHTML 1.0:
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

HTML Form

Form tag Properties

  1. action="" specifies the to where should data be submitted (default: ).
  2. method="" specifies the method (get/post) when submitting data (default: get).
  3. target="" specifies view location when perform the action (default: _self).
  4. accept-charset="" specifies charset when submitting data.
  5. enctype="" specifies codec when submitting data (default: url-encoded).
  6. autocomplete="" specifies on or off of auto completion.
  7. novalidate="" specifies on or off of validation of form.
